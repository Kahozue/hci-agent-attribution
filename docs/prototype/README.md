## 修改重點與 HCI 理由

| 面向 | v1 問題 | v2 改動 | HCI 理由 |
|---|---|---|---|
| 版面層級 | 過度扁平、細線分區，主次不分 | 卡片容器＋細緻陰影、明確字級與間距節奏 | 降低 Gulf of Evaluation，讓受試者快速分辨任務／輸出／作答區 |
| 進度回饋 | 僅文字「第 N 題」 | 進度條＋情境 pill | Situation awareness：清楚知道進度與當前情境框 |
| 作答元件 | 可點 `span`，**無法鍵盤操作** | 改為真正的 `<button>`、`aria-pressed` 標選取 | 可存取性（鍵盤、螢幕報讀），選取態更明確 |
| 工具序列 | 純文字 | 等寬字 chip | 提升可掃描性，凸顯「工具路徑」這個歸因線索 |
| 揭露結果 | 單色提示 | 語意色（正確綠／不正確琥珀）＋文字雙重標示，非僅靠顏色 | 即時回饋且不依賴顏色獨撐，支援信任校準 |
| 響應式 | 無 | 手機單欄 | 受試者裝置相容 |

設計約束：繁體中文、無 emoji、無與內容無關的裝飾、無細小字體與符號；視覺處理一律服務於資訊層級與可讀性。
