名詞解釋

任務
兩個 agent 都在處理同一個 coding task，例如 bug 修復、補測試、加入 logging 或 benchmark。畫面會給任務類型與情境，不只顯示 task id。
M1–M4 一致度
四種歸因方法對此 pair 的判斷一致程度。4/4 代表四種方法同意；摘要版不顯示此欄，證據版才顯示。
trace 出處
此輸出背後的執行紀錄檔路徑，用來支持來源可追蹤性。它不是要受試者讀 JSON，而是標示證據來源。
判斷類型
資料建立時主要依據的差異線索，例如初始工具策略、輸出慣例、任務成敗落差或重跑變異。
A/B 呈現
A 是摘要版，B 是證據版。兩者比較的是透明性與 progressive disclosure 對歸因判斷的影響。
